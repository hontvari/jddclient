Administration

Ubuntu (Debian etc.)

The log file is located in /var/log/jddclient. If you filled the mail configuration section in the logback-daemon.xml configuration file, then you will also receive error messages in mail.

The name of the daemon is jddclient. For example to restart the daemon enter this command:
sudo service jddclient restart.

If an error occurs when jddclient sends the new IP address to the dynamic DNS service provider, then depending on the type of the error jddclient may disable the corresponding updater. This prevents the disabling of the account by the provider. Some provider disable an account if repeated invalid updates occur. In this case examine the logfile, correct the problem in the configuration and reset the updater. Resetting the updater is done by forcing an update. Enter the following command on the terminal:
jddclient --force

You can also use the jddclient command to manually do an update. The jddclient command without parameters check the IP address, and if it is changed, then sends it to the corresponding dynamic DNS providers.

For more information about the jddclient, use the --help option:
jddclient --help

jddclient stores the current status of the updaters in the /var/lib/jddclient/status.xml file. You should not edit this file, but you may want to look at it. If this file becomes corrupt, and --force does not help, then after stopping the dameon, you can simply delete it. It will be regenerated when you start the daemon again.

Other operating systems

The administration is similar to what is described in the Ubuntu section above, but depends on how you actually setup jddclient.

Basically you run jddclient with this command:
java -jar <path-to-jddclient.jar>
For example to dipslay the command line options enter this command on the terminal / command prompt:
java -jar <path-to-jddclient.jar> --help

On Windows the the full path of the status file is %ALLUSERSPROFILE%\jddclient\status.xml.
